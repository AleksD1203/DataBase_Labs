**ФАЙЛ 1**: aggregation_queries.sql
Запит 1: Загальна кількість клієнтів
Що робить: Підраховує загальну кількість записів у таблиці CUSTOMER.
Навіщо: Для отримання базової метрики - скільки клієнтів зареєстровано в системі.
Функція: COUNT(*)

Запит 2: Загальна сума всіх бронювань
Що робить: Обчислює суму всіх значень у стовпці total_amount таблиці BOOKING.
Навіщо: Для визначення загального доходу від усіх бронювань.
Функція: SUM()

Запит 3: Середня тривалість фільмів
Що робить: Розраховує середнє арифметичне тривалості всіх фільмів.
Навіщо: Для аналізу середньої довжини фільмів у прокаті.
Функція: AVG()

Запит 4: Максимальна та мінімальна вартість квитка
Що робить: Знаходить найвищу та найнижчу ціну квитка.
Навіщо: Для визначення діапазону цін на квитки в кінотеатрі.
Функції: MAX(), MIN()

Запит 5: Кількість сеансів всього
Що робить: Підраховує загальну кількість сеансів.
Навіщо: Для аналізу завантаженості кінотеатру.
Функція: COUNT(*)

Запит 6: Середня ціна квитка
Що робить: Обчислює середню ціну всіх проданих квитків.
Навіщо: Для аналізу середньої вартості квитка.
Функція: AVG()

Запит 7: Мін/макс кількість місць у залах
Що робить: Знаходить зал з найменшою та найбільшою місткістю.
Навіщо: Для планування розподілу фільмів по залах.
Функції: MIN(), MAX()

**ФАЙЛ 2:** group_by_queries.sql
Запит 2: Кількість сеансів для кожного фільму
Що робить: Групує сеанси за фільмами та підраховує кількість сеансів для кожного.
Навіщо: Для визначення популярності фільмів (які фільми частіше показують).
Групування: GROUP BY m.title

Запит 2: Загальна сума бронювань для кожного клієнта
Що робить: Групує бронювання за клієнтами та підсумовує їх витрати.
Навіщо: Для аналізу лояльності клієнтів та їх витрат.
Групування: GROUP BY c.name

Запит 3: Кількість місць у кожному залі
Що робить: Виводить список залів з їх місткістю у порядку спадання.
Навіщо: Для порівняння місткості різних залів.
Сортування: ORDER BY total_seats DESC

Запит 4: Кількість квитків, проданих на кожен фільм
Що робить: Групує продані квитки за фільмами.
Навіщо: Для аналізу касових зборів по фільмах.
Групування: GROUP BY m.title

Запит 5: Кількість бронювань по днях
Що робить: Групує бронювання за датами їх створення.
Навіщо: Для виявлення днів з найвищою активністю бронювань.
Групування: GROUP BY booking_date

**ФАЙЛ 3**: having_queries.sql
Запит 1: Клієнти, які витратили більше 200 грн
Що робить: Вибирає клієнтів, сумарні витрати яких перевищують 200 грн.
Навіщо: Для визначення найбільш цінних клієнтів (VIP).
Фільтрація груп: HAVING SUM(b.total_amount) > 200

Запит 2: Жанри з середньою тривалістю > 140 хв
Що робить: Вибирає жанри фільмів, середня тривалість яких більше 140 хвилин.
Навіщо: Для аналізу тривалості фільмів за жанрами.
Фільтрація груп: HAVING AVG(m.duration) > 140

Запит 3: Зали з більше ніж 1 сеансом
Що робить: Знаходить зали, в яких заплановано більше одного сеансу.
Навіщо: Для виявлення найбільш завантажених залів.
Фільтрація груп: HAVING COUNT(s.session_id) > 1

Запит 4: Фільми з більше ніж 1 проданим квитком
Що робить: Вибирає фільми, на які продано більше одного квитка.
Навіщо: Для визначення популярних фільмів.
Фільтрація груп: HAVING COUNT(t.ticket_id) > 1

Запит 5: Дні з більше ніж 1 бронюванням
Що робить: Знаходить дні, коли було зроблено більше одного бронювання.
Навіщо: Для виявлення пікових днів продажів.
Фільтрація груп: HAVING COUNT(*) > 1

**ФАЙЛ 4**: join_queries.sql
Запит 1: INNER JOIN - детальна інформація про квитки
Що робить: Об'єднує 6 таблиць для відображення повної інформації про кожен квиток.
Навіщо: Для отримання детального звіту по всіх проданих квитках.
Тип JOIN: Множинний INNER JOIN

Запит 2: LEFT JOIN - всі зали та їх сеанси
Що робить: Показує всі зали, навіть ті, в яких ще не було сеансів.
Навіщо: Для перегляду всіх залів незалежно від їх зайнятості.
Тип JOIN: LEFT JOIN

Запит 3: RIGHT JOIN - всі сеанси та зали
Що робить: Показує всі сеанси з інформацією про зали.
Навіщо: Для аналізу розкладу сеансів.
Тип JOIN: RIGHT JOIN

Запит 4: FULL JOIN - всі клієнти та всі бронювання
Що робить: Показує всіх клієнтів і всі бронювання, включаючи незаповнені відповідності.
Навіщо: Для виявлення клієнтів без бронювань та бронювань без клієнтів.
Тип JOIN: FULL JOIN

Запит 5: CROSS JOIN - всі комбінації фільмів та залів
Що робить: Створює декартовий добуток фільмів та залів.
Навіщо: Для планування можливих розкладів сеансів.
Тип JOIN: CROSS JOIN

**ФАЙЛ 5**: subquery_queries.sql
Запит 1: Підзапит у SELECT - кількість бронювань клієнта
Що робить: Для кожного клієнта додає стовпець з кількістю його бронювань.
Навіщо: Для аналізу активності клієнтів без групування.
Підзапит: У вибірковому списку SELECT

Запит 2: Підзапит у WHERE - квитки дорожчі за середню ціну
Що робить: Вибирає квитки, ціна яких вища за середню ціну всіх квитків.
Навіщо: Для виявлення "преміальних" квитків.
Підзапит: У умові WHERE

Запит 3: Підзапит у HAVING - жанри з тривалістю вище середньої
Що робить: Вибирає жанри, де середня тривалість фільмів перевищує загальну середню.
Навіщо: Для порівняння жанрів із середніми показниками.
Підзапит: У умові HAVING

Запит 4: Підзапит у FROM - найпопулярніші фільми
Що робить: Використовує підзапит як тимчасову таблицю для подальшого фільтрування.
Навіщо: Для виконання складних обчислень у кілька етапів.
Підзапит: У реченні FROM

Запит 5: Підзапит з IN - клієнти з бронюваннями
Що робить: Вибирає клієнтів, які мають хоча б одне бронювання.
Навіщо: Для фільтрації активних клієнтів.
Підзапит: З оператором IN

**ФАЙЛ 6**: multitable_aggregation.sql
Запит 1: Загальний дохід від кожного фільму
Що робить: Об'єднує 3 таблиці та агрегує дохід по фільмах.
Навіщо: Для аналізу касових зборів по кожному фільму.
Складність: 2 JOIN + GROUP BY

Запит 2: Кількість проданих квитків на кожен сеанс
Що робить: Показує детальну статистику по кожному сеансу.
Навіщо: Для аналізу відвідуваності окремих сеансів.
Складність: 3 JOIN + GROUP BY

Запит 3: Загальна виручка по залах
Що робить: Агрегує виручку по кожному залу.
Навіщо: Для оцінки ефективності використання залів.
Складність: 2 JOIN + GROUP BY

Запит 4: Доход по днях з деталізацією
Що робить: Групує дохід за датами та фільмами.
Навіщо: Для денного аналізу продажів з деталізацією.
Складність: 2 JOIN + GROUP BY за двома полями

Запит 5: Статистика по клієнтах з деталізацією
Що робить: Надає комплексну статистику по кожному клієнту.
Навіщо: Для всебічного аналізу клієнтської бази.
Складність: 2 LEFT JOIN + GROUP BY
